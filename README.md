# Terracraft Valrising Bot

Программа моделирует работу бота для подземелья из задания Terracraft Valrising.

Программа читает описание подземелья из входного файла и записывает результат работы бота в файл `result.txt`.

## Требования
Для сборки проекта нужны:

- CMake
- C++ compiler с поддержкой C++17 или выше
- Git

Проект использует GoogleTest как git submodule.

## Клонирование проекта

Склонируйте репозиторий:

```bash
git clone <repository-url>
cd Terracraft-Valrising-bot
```

Если проект клонируется впервые, нужно загрузить submodule:

```bash
git clone --recurse-submodules <repository-url>
```

## Если проект уже склонирован без submodule:
```bash
git submodule update --init --recursive
```

## Сборка проекта

Из корня проекта:

```bash
cmake -S . -B build
cmake --build build
```

## Запуск программы

Программа принимает путь к входному файлу как аргумент.

Пример:
```bash
./build/task in.txt
```
После запуска результат будет записан в файл:

result.txt

Посмотреть результат можно так:
```bash
cat result.txt
```

## Запуск тестов

После сборки проекта тесты можно запустить командой:
```bash
./build/tests/terracraft_tests
```

Пример полного запуска
```bash
cmake -S . -B build
cmake --build build
./build/tests/terracraft_tests
./build/task in.txt
cat result.txt
```

## Замечания к примеру из задания

### Восстановление обратных рёбер

В реализации граф подземелья считается неориентированным.
В примере из задания есть ребро из комнаты `3` в комнату `4`:

```txt
3 1,4 1 0 2 40
```
но в описании комнаты 4 обратное ребро на комнату 3 не указано:
```txt
4 2,5 2 4 0 15
```
При этом в ожидаемом выводе бот переходит из комнаты 4 обратно в комнату 3
Поэтому при чтении входных данных программа автоматически восстанавливает обратные рёбра: если есть проход A -> B, но нет прохода B -> A, он добавляется. После этого списки соседей сортируются по возрастанию.

## Расчёт итоговой ценности

В примере задания итоговая ценность указана как 93, однако при указанных ценностях ресурсов это значение не получается. 
Для собранных ресурсов gold = 4 и gems = 3:
- без удвоения gems: 4 * 11 + 3 * 23 = 113;
- с удвоением gems, как указано в условии: 4 * 11 + 3 * 46 = 182.

В программе используется второй вариант, так как он соответствует тексту условия.
